## TL;DR

LangMem splits agent memory into two complementary modes: memory tools the agent can use during active interaction, and a background manager that consolidates longer-term knowledge outside the hot path.

## Key Claims

- Long-term memory needs both interactive tooling and background maintenance.
- Agent memory is not only vector retrieval; it also includes extraction, consolidation, prompt refinement, and profile updates.
- Memory tooling should work with any storage backend while also integrating tightly with a runtime's native store when available.
- Production memory systems need persistence beyond in-memory conversation state.

## Important Details

- LangMem presents a core memory API that works with arbitrary storage systems.
- The docs explicitly distinguish hot-path memory tools from a background memory manager.
- The memory manager is described as extracting, consolidating, and updating agent knowledge over time.
- The quickstart uses `create_manage_memory_tool` and `create_search_memory_tool`, making memory itself part of the tool surface.
- The docs recommend DB-backed persistence such as Postgres-backed stores for production instead of in-process memory.
- LangMem is also positioned as a natural companion to LangGraph's long-term memory store rather than a standalone theoretical memory idea.
